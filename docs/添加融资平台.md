
1) 角色与权限模型（最关键）
A. 企业用户（Issuer / Company）
* 注册/登录
* 选择服务等级（按照管理员设置的会员服务类型选择）
* 填写企业信息、融资信息
* 上传 BP（可选：补充财务/数据室链接）
* 设置上传信息“是否对投资人可见”“可见范围”
* 查看自己提交状态、管理员反馈
B. 管理员（Admin）
* 查看全部企业信息
* 审核/编辑/加标签/内部备注
* 控制“是否对投资人可见”“可见范围”
* 设置对投资人的服务等级和服务内容
* 导出数据（CSV/Excel）
* 管理投资人账号/权限（可选）
C. 投资人（Investor）
* 注册/登录（建议邀请制或审核制）
* 浏览已审核且授权可见的企业项目
* 搜索/筛选（行业、阶段、融资规模、地区等）
* 发起“请求引荐/联系”（建议不要直接看到企业联系方式，先走你这边）
建议默认：投资人只能看到脱敏版企业信息，联系方式/全量 BP 需要“请求权限/引荐”。这是保护企业、也保护你商业模式的核心。

2) 核心流程（MVP）
流程 1：企业提交
1. 企业注册/登录
2. 填写表单（企业信息 + 出海/结构要点 + 融资要素）
3. 上传 BP（PDF/PPT/PPTX）
4. 提交 → 状态变为「待审核」
流程 2：管理员审核与上架
1. Admin 在后台看到新提交
2. 校验信息完整性、合规风险（敏感数据、受控技术、主体结构等）
3. 打标签/内部备注
4. 设定可见性：
    * 不可见（仅内部）
    * 对投资人可见（脱敏）
    * 对指定投资人可见（白名单）
5. 通过后状态变为「已上架」
流程 3：投资人浏览与申请引荐
1. Investor 登录后看到项目列表（仅可见项目）
2. 点开项目详情（默认脱敏）
3. 点击「请求 BP / 请求引荐」
4. Admin 收到请求 → 决定：批准/拒绝/需要补充
5. 批准后：
    * 可开放 BP 下载权限（限时/水印/不可转发可选）
    * 或由你人工发起邮件引荐（更符合你2B信任模型）

3) 页面与模块清单（网站信息架构）
企业端（Company Portal）
* 注册/登录
* 企业资料（Company Profile）
* 融资信息（Fundraising）
* BP/资料上传（Documents）
* 提交记录与状态（Submissions & Status）
* 消息中心（Admin feedback / Investor requests）
投资人端（Investor Portal）
* 注册/登录（建议“申请 + 审核”）
* 项目列表（Deal Flow）
* 项目详情页（Deal Detail）
* 请求引荐 / 请求资料（Request Access）
* 我的请求记录（My Requests）
管理后台（Admin Console）
* 企业列表 + 筛选
* 企业详情 + 审核
* 可见性与权限管理（Visibility / ACL）
* 投资人列表 + 审核
* 请求管理（Requests）
* 标签/行业/阶段字典管理
* 导出报表

4) 数据字段（MVP 必要字段）
企业信息（Company）
* 公司名称（中/英）
* 官网/LinkedIn（可选）
* 行业（主/副）
* 所在地（总部/研发）
* 一句话产品简介
* 阶段（Seed/A/B/上市等）
* 牵头联系人（姓名/职务/邮箱/WhatsApp/微信）
融资信息（Fundraising）
* 融资目的（出海建厂/并购/营运资金等）
* 融资类型偏好（股权/债/项目融资/RWA）
* 融资金额区间
* 时间窗口（0-3/3-6/6-12个月）
* 估值预期（可选）
* 资金用途（Use of Funds）
* 海外结构现状（是否有 SG/HK/US 主体）
* IP 归属/可迁移性（选择+备注）
* 数据敏感性（用于合规风险预筛）
文档（Documents）
* BP 文件（必选）
* 数据室链接（可选）
* 财务摘要（可选，后续迭代）
* 提供企业的服务等级和服务内容

5) 访问控制与脱敏策略（一定要做）
默认：投资人不可见企业联系方式
* 投资人看到：公司简介、行业、阶段、融资需求、市场计划、结构状态（概述）
* 投资人看不到：联系人电话/微信、敏感客户名单、详细财务、全量 BP（除非授权）
